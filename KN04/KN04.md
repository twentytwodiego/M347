## Was macht `docker compose up`?
 
`docker compose up` ist eine Sammelaktion aus mehreren Schritten:
 
1. Netzwerk(e) und Volume(s) aus der Compose-Datei erstellen (falls nicht vorhanden)
2. Images laden (`pull`), falls lokal nicht vorhanden
3. Images bauen (`build`), falls ein `build`-Abschnitt definiert ist
4. Container mit der Compose-Konfiguration erstellen (`create`)
5. Container starten (`start`)
6. Logs im Vordergrund anzeigen (ohne `-d`)
 
Praktisch ähnlich zu:
 
```powershell
docker compose pull
docker compose build
docker compose create
docker compose start
docker compose logs -f
```

### Warum gibt es bei `db.php` einen Fehler?
 
In `db.php` steht als Datenbank-Host hart codiert `kn02b-db`.
 
In Teil B heisst der DB-Container aber `m347-kn04b-db` und es gibt keinen Alias `kn02b-db`. Dadurch kann der Hostname nicht aufgelöst werden und die MySQL-Verbindung scheitert.
 
### Wie löst man den Fehler?
 
Möglichkeit 1:
 
- `db.php` anpassen und als Host den Compose-Service-Namen (z. B. `db`) verwenden.
 
Möglichkeit 2:
 
- In Compose beim DB-Service einen Netzwerk-Alias `kn02b-db` setzen.
